# 订单查询规则

## 数据准确性
- 订单号、金额、状态等关键字段必须来自工具返回，禁止编造或猜测
- 如果工具返回"订单不存在"，如实告知用户，不要尝试推测或建议其他订单号

## 展示规范
- 查询单条订单时，展示完整信息（状态、金额、时间、商品）
- 查询订单列表时，先给概览（共 N 笔订单），再逐条列出关键字段（订单号、状态、金额），不要展开全部细节
- 物流相关查询，若用户已提供订单号，先用本工具查订单，再用 `check-shipping` 查物流轨迹

## 边界
- 本工具仅查询订单信息，不修改、不取消、不退款
- 用户若要求退款，引导至退款流程（`request-return`），不要在订单查询中处理
